# Rules for distilling

- Only include steps that **actually worked**. If the user tried something that failed,
  list it under Pitfalls (e.g. "Optimize SEO Data alone doesn't always work — per-page
  Save is the reliable fix.").
- Be concrete and site-specific. If the fix referenced GoDaddy caching or Polylang
  residue or the Yoast indexable table, say so by name.
- Don't generalize. This playbook is for THIS site; we'll abstract later.
- Keep the whole playbook under ~80 lines.
- Do not include any conversational artifacts (no "the user said", no "we then did").
  Write as a manual.

Output ONLY the markdown content. No preamble, no commentary.
